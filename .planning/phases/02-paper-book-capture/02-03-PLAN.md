  <name>task 1: Implement image quality detection</name>
  <files>src/core/ocr/postprocess.rs</files>
  <action>
    Add quality detection functions:
    
    1. `calculate_quality_score(image_data: &[u8]) -> Result<f32>`:
       - Decode image
       - Calculate blur score using Laplacian variance (low variance = blurry)
       - Calculate brightness (mean pixel value)
       - Return combined score 0.0-1.0
    
    2. `should_retry(ocr_result: &OcrResult) -> bool`:
       - Return true if overall confidence < 0.5
       - Return true if any critical region has confidence < 0.3
       - Return false otherwise
    
    3. Thresholds (tunable):
       - Blur threshold: Laplacian variance < 100 = blurry
       - Brightness: mean < 50 or > 200 = too dark/bright
       - OCR confidence: < 0.5 = low confidence retry
    
    Use image crate for processing. Keep algorithms simple and fast.
  </action>
  <verify>
    <automated>cargo test ocr::postprocess::tests -- --nocapture</automated>
  </verify>
  <done>
    - calculate_quality_score returns 0.0-1.0 score
    - should_retry correctly identifies low-confidence results
    - Tests with sample images (blurry, dark, normal)
    - Processing time < 100ms for quality check
  </done>

    Make OCR run in parallel with auto-retry:
    
    1. In run_ocr handler, use spawn for parallel execution:
       ```rust
       spawn(async move {
           let mut result = ocr_engine.process_image(&image_data).await?;
           
           // Auto-retry if low confidence
           if should_retry(&result) {
               // Retry with different parameters (e.g., different threshold)
               result = ocr_engine.process_image_with_params(&image_data, retry_params).await?;
           }
           
           // Send result back to UI via signal
           ocr_result.set(Some(result));
       });
       ```
    
    2. Add retry logic:
       - First attempt: default parameters
       - Retry 1: lower confidence threshold, more aggressive preprocessing
       - Retry 2: (optional) different binarization method
       - Max 2 retries to prevent infinite loops
    
    3. Update UI state after each attempt
    4. Show "Retrying..." message during retry attempts
    
    5. Add error handling for timeout (> 10s = show timeout error)
  </action>
