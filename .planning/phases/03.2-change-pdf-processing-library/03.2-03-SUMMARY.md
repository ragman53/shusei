## Implementation Details

### Task 1: Error Mapping

**Status:** Not applicable - hayro doesn't export `PdfError` or `LoadPdfError` types publicly.

**Current approach:** Inline error handling in `pdf.rs` converts hayro errors to `ShuseiError::Io` with descriptive messages:

```rust
let pdf = Pdf::new(std::sync::Arc::new(pdf_data.clone())).map_err(|e| {
    ShuseiError::Io(std::io::Error::new(
        std::io::ErrorKind::InvalidData,
        format!("Failed to load PDF: {:?}", e),
    ))
})?;
```

This approach is sufficient for the use case and doesn't require a `From` implementation.

### Task 2: Retry Logic

**Status:** Complete - retry-once pattern already implemented in previous plan (03.2-02).

**Verified implementation:**

1. **`render_with_retry()` helper** (line 88-104):
   - Attempts initial render
   - Logs warning on failure
   - Retries once before returning error

2. **`render_all_pages()` updated** (line 119):
   - Now uses `render_with_retry()` instead of `render_page()`

3. **`render_pages_batch()` skip-on-failure** (line 164-179):
   - Uses `filter_map` to skip failed pages after retry
   - Logs warning for each skipped page
   - Continues processing remaining pages
   - Returns partial results

**Error logging format:**
```rust

```
