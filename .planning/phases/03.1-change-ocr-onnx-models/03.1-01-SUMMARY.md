## Key Decisions Made

1. **Backup strategy:** Old models renamed to .bak instead of deleted
   - Rationale: Allows quick rollback if NDLOCR-Lite has issues
   - Trade-off: Temporary disk space usage (165MB backup)

2. **Recognition model selection:** Chose medium-sized PARSeq model
   - File: `parseq-ndl-16x384-50-tiny-146epoch-tegaki2.onnx`
   - Rationale: Good balance of speed and accuracy for Japanese text
   - Alternative: Smaller model available for speed-critical scenarios

3. **Dictionary removal:** PARSeq model has internal tokenizer
   - No external dict.txt needed
   - Simplifies deployment and reduces file count
