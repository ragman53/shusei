# Research: NDLOCR-Lite Model Migration

**Phase:** 03.1-change-ocr-onnx-models  
**Date:** 2026-03-12  
**Confidence:** HIGH

## Summary

Switch from PaddleOCR ONNX models to NDLOCR-Lite models for Japanese OCR. NDLOCR-Lite is developed by National Diet Library (NDL) Japan, optimized for Japanese text including classical books, and runs efficiently on CPU without GPU.

## Model Comparison

| Aspect | PaddleOCR (Current) | NDLOCR-Lite (Target) |
|--------|---------------------|----------------------|
| Detection Model | `text_detection.onnx` (84MB) | `deim-s-1024x1024.onnx` (39MB) |
| Recognition Model | `text_recognition.onnx` (81MB) | `parseq-ndl-16x384-50-tiny-146epoch-tegaki2.onnx` (~20MB est.) |
| Dictionary | `dict.txt` (73KB, Chinese) | Built-in PARSeq tokenizer |
| Total Size | ~165MB | ~60MB (63% reduction) |
| Input Size | 960x960 | 1024x1024 (detection), variable (recognition) |
| Language Support | Chinese/Japanese | Japanese optimized |
| License | Apache 2.0 | CC BY 4.0 |
| Source | HuggingFace monkt/paddleocr-onnx | GitHub ndl-lab/ndlocr-lite |

## Key Technical Differences

### 1. Detection Model Output Format

**PaddleOCR:**
- Output shape: [1, num_boxes, 4] or [1, num_boxes, 5]
- Format: [x1, y1, x2, y2, confidence?]
- Coordinate system: Normalized or pixel

**NDLOCR-Lite (DEIMv2):**
- Output shape: Likely similar but needs validation
- Uses DINOv3-based detection (newer architecture)
- May have different postprocessing requirements

### 2. Recognition Model Architecture

**PaddleOCR:**
- CRNN-style architecture
- Requires external dictionary (dict.txt)
- CTC decoding with character index lookup

**NDLOCR-Lite (PARSeq):**
- Autoregressive sequence model
- Self-contained tokenizer (no external dict.txt needed)
- May use different decoding strategy

### 3. Input Preprocessing

**PaddleOCR:**
- Input: 960x960 grayscale
- Normalization: [0, 1]

**NDLOCR-Lite:**
- Detection: 1024x1024 RGB (3 channels)
- Recognition: 32px height, variable width
- May require different normalization

## Download Script

```bash
#!/bin/bash
# Download NDLOCR-Lite models

MODEL_DIR="assets/ocr/models"
mkdir -p "$MODEL_DIR"

# Detection model
echo "Downloading detection model (39MB)..."
curl -L -o "$MODEL_DIR/deim-s-1024x1024.onnx" \
  "https://github.com/ndl-lab/ndlocr-lite/raw/master/src/model/deim-s-1024x1024.onnx"

# Recognition model (medium size, good balance)
echo "Downloading recognition model..."
curl -L -o "$MODEL_DIR/parseq-ndl-16x384-50-tiny-146epoch-tegaki2.onnx" \
  "https://github.com/ndl-lab/ndlocr-lite/raw/master/src/model/parseq-ndl-16x384-50-tiny-146epoch-tegaki2.onnx"

# Backup old models
echo "Backing up old PaddleOCR models..."
mv "$MODEL_DIR/text_detection.onnx" "$MODEL_DIR/text_detection.onnx.bak" 2>/dev/null || true
mv "$MODEL_DIR/text_recognition.onnx" "$MODEL_DIR/text_recognition.onnx.bak" 2>/dev/null || true
mv "$MODEL_DIR/dict.txt" "$MODEL_DIR/dict.txt.bak" 2>/dev/null || true

echo "Done! Models downloaded to $MODEL_DIR"
ls -lh "$MODEL_DIR"/*.onnx
```

## Risks and Mitigation

| Risk | Likelihood | Impact | Mitigation |
|------|------------|--------|------------|
| Model output format differs | Medium | High | Test with sample image, adjust postprocessing |
| PARSeq requires different decoding | Medium | High | Review PARSeq documentation, may need tokenizer integration |
| Models don't load on Android | Low | High | Test on Android device early, verify ONNX opset compatibility |
| Japanese accuracy worse than PaddleOCR | Low | Medium | Benchmark with sample pages, keep PaddleOCR as fallback |
| Model files too large for bundling | Low | Medium | Use download-on-first-run strategy |
