<domain>
## Phase Boundary

Human verification of PDF loading functionality on real Android devices. This phase validates the hayro integration and Load Demo PDF feature that was built in Phase 03.2 and Quick Tasks 004/006.

**In scope:**
- Test PDF loading on real Android devices (not emulator)
- Verify Load Demo PDF button functionality
- Validate OCR processing with test PDF
- Document any issues and required fixes
- Ensure no crashes or memory issues

**Out of scope:**
- New feature development
- Code changes (unless fixes are needed)
- Desktop testing (covered by quick task 006)

</domain>

<decisions>
## Implementation Decisions

### Verification Scope
- **Target devices:** Real Android devices (priority: mid-range 2-4GB RAM)
- **Test PDF:** Bundled demo PDF (`assets/test/medium_pdf_test.pdf`)
- **Test cases:** Load Demo PDF button, PDF import, OCR processing, UI responsiveness

### Success Criteria
- **No crashes:** App launches and PDF loads without crashing
- **OCR works:** OCR processes pages and shows extracted text
- **UI responsive:** UI remains usable during processing
- **Memory stable:** No OOM on 2GB RAM devices

### OpenCode's Discretion
- Specific test steps and documentation format
- Issue categorization and priority assignment
- Fix recommendations for discovered issues

</decisions>

<code_context>
## Existing Code Insights

### What Was Built (Phase 03.2)
- `src/core/pdf.rs` - PdfProcessor with hayro integration (706 lines)
- hayro pure Rust PDF rendering (no FFI, eliminates CRT conflict from pdfium)
- Batch processing with 10 pages per batch
- Parallel rendering with 3 threads using rayon
- Error handling with retry-once logic

### What Was Built (Quick Tasks)
- `assets/test/medium_pdf_test.pdf` - Bundled test PDF
- `src/platform/android.rs` - `copy_asset_to_files()` JNI function
- `src/ui/library.rs` - Load Demo PDF button (orange, Android-only)
- `MainActivity.java` - `copyAssetToFiles()` static method

### Known Issues
- **CRT conflict from ort dependency:** Tests cannot compile (pre-existing, not from hayro)
- **PDF metadata limited:** hayro doesn't expose PDF metadata yet, only page count

### Integration Points
- `PdfProcessor::import_pdf()` - Entry point for PDF import
- `PdfConversionService::convert_pdf()` - Full PDF → OCR conversion
- `load_demo_pdf` handler in `src/ui/library.rs` - Button handler

</code_context>

<specifics>
## Specific Ideas

### Test Procedure
1. Build and deploy APK to real Android device
2. Launch app and navigate to Library screen
3. Tap "Load Demo PDF" button (orange)
4. Verify PDF metadata dialog appears
5. Start OCR processing
6. Monitor for crashes, memory issues, UI responsiveness
7. Verify extracted text is readable

### Expected Behavior
- PDF loads within 1-2 seconds
- OCR processes pages progressively with progress indicator
- UI remains responsive during processing
- No crashes or ANR (Application Not Responding) dialogs

</specifics>
